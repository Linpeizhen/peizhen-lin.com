KKday Official Website (Desktop)

KKday 電腦版官網是我成為 UI 設計師後第一個負責的產品，兩年多來陸續統整整站的視覺風格和操作邏輯，並建立了 Design Guidelines。 除了設計之外，我也負責靜態頁面切版，用 Git 和前端工程師協作。
這個產品的團隊由各種不同職能的角色組成，採取敏捷式開發，短週期的固定會議讓開發和溝通上更有效率，也讓團隊成員有更多機會互相討論。
我們也會對新 feature 進行易用性測試，由 UX designer 進行訪談，團隊成員擔任觀察員。另外定期有 Brainstorming 會議，讓大家提出自己的想法。
我很幸運能在剛入行就能加入這個團隊，在這個團隊裡我學到了很多產品開發的觀念、技術和溝通能力。例如一開始官網的開發是採用瀑布式流程，開發時程較長，溝通效率低，曾發生專案開發到後來需求要因應市場變更，造成產出結果不盡理想。 後來組織改動，官網改用敏捷式開發，才開始有大家一起共同為「產品」努力的感覺，也讓我學習到很多敏捷開發的精神。

我負責的項目

  • 流程規劃
  • 介面設計
  • 靜態頁面切版

合作夥伴：

  • Product owner
  • UX and UI designer
  • Front-end and Back-end developers
  • Business development and Marketing specialists

STARLUX 考核系統

這個專案是將舊有的考核系統全面改版，此系統有許多複雜的流程和操作，但藉由 Flow chart 的整理和 Design Guidelines 的建立，我成功將流程化繁為簡，並創造友善的使用者體驗和舒適的閱讀體驗。
其中年度考核表是最複雜的一環，牽涉到欄位的填寫權限、人員權限不同造成的畫面顯示差異以及來來回回的簽核流程，最有挑戰的是如何讓複雜的功能和流程狀態讓使用者易於理解。 例如完整的考核共有 7 個步驟，原需求書並無提供任何說明讓使用者理解當下所在的步驟，在設計過程中我發現沒有提供流程狀態的指示，會無法理解要進行什麼動作，於是我在考核表下方加上當前狀態的提示，並搭配考核表頁籤的鉛筆圖標，讓使用者了解該填寫哪些頁面及現在考核的狀態。

我負責的項目

  • 流程規劃 / Flow chart
  • 介面設計
  • 靜態頁面切版
  • 建立 Design Guidelines

合作夥伴：

  • Project Manager
  • Front-end developers
  • Back-end developers
  • Human Resources Specialist / Manager

這個專案是由我自身經驗所衍伸出來的發想，我正在進行背包輕量化，但找不到一個適合我的裝備管理應用，於是決定自己動手做。
一開始的需求都由我和夥伴自己提出，已經有初步的設計稿和功能可以使用，但如果想要推廣這個產品，讓其他戶外活動者也可以使用，勢必要進行一些使用者研究和訪談，來了解目前戶外活動者對於裝備管理的需求和痛點，以更加確定此產品的方向。目前使用者研究正在規劃中。
