星宇航空
員工考核系統

星宇航空內部使用的考核系統，原先只有試用期考核的功能，而年度考核皆由人資單位人工處理，包含製作 Excel 考核表、回收考核表、統計考核結果、發送 Email 通知等。

因應員工數快速上升，考核系統欲將年度考核功能納入開發規劃，讓年度考核過程系統化、數位化，減少人資的人力負擔，我也透過設計讓介面和流程簡單易懂，藉此減少員工和人資單位的來回溝通。

  • 繪製年度考核流程的 Flow chart

  • HTML 切版，並使用 Git 交付給前端工程師套版
  • 模組化元件並製作後台版型的 Design Guidebook
  • 參與需求單位 ( 人資 ) 的測試
  • 試用考核功能介面優化

我做了什麼

1. 重新定義考核系統的 Sitemap

先前的資訊架構只考慮到試用考核，因此在「HR Dashboard」的兩個子層是「試用考核列表」以及「設定」，而設定包含試用考核設定和系統設定。年度考核功能加入後，HR dashboard 也需要新增年度考核的功能，若直接套入這個架構，會造成資訊層級不夠明確。

另外，HR 同仁在做設定時，通常一次只會專注在其中一個功能，例如年度考核一年設定兩次、試用考核較少異動，系統設定不規律但也不頻繁，且三者通常不會需要同時操作，所以我根據這個情境做了一些調整。

△ 舊版 sitemap

調整後：功能擴大且結構清晰的考核系統

我把原本的「HR Dashboard」分成三個次頁：「試用考核設定」、「年度考核設定」以及「系統設定」，讓 HR 同仁可以更專注在其中一個功能設定。

第一版修正：「更明確、簡潔的搜尋體驗」

為了讓搜尋功能更明顯，且不打斷使用者瀏覽，我們決定在首頁直接放上搜尋文字框，而我透過卡片式設計讓搜尋區塊更突出。另外，我們也改用下拉選單呈現推薦結果。

我們還發現，大部分使用者是透過打字搜尋欲前往的城市進入商品列表頁，而不是從城市選單點擊，所以我們將原本多層的城市選單移除。

第二版修正：「更清晰、便利的搜尋體驗」

第一版優化上線之後，我們仍持續追蹤首頁的使用者數據，我們發現，搜尋欄位的點擊數遠大於下方城市、商品，由此得知大多數使用者進到 KKday，都是以搜尋為主要導航工具，鮮少透過瀏覽的方式進行探索。我們推測原因是大部分使用者都是已經決定要到某地旅行才到 KKday，在有明確目的地的情況下，自行搜尋是最快的方式，因此我們決定將搜尋和行銷活動區塊分開，減少視覺干擾，讓使用者一進入首頁可以更明確看見搜尋欄位，且將使用者最近瀏覽的幾個商品也加入首頁，讓使用者不需要回憶並搜尋之前瀏覽過的商品，減少記憶負擔。

在 Banner 的設計上，我選用了幾張城市照片當作底圖，並且以右側擺放單一主角的構圖為主，以突出左側的搜尋區域。

最近瀏覽、行銷 Banner、城市和推薦商品卡片，皆使用不同的卡片版型，以分隔出三者的差異，讓使用者更清楚知道有不同類型的資訊。

2. 提供城市內搜尋，縮小搜尋範圍

過往是無法只搜尋單一城市的商品，但根據上述的發現，大部分使用者都有明確的目的地，此時在城市商品列表頁進行關鍵字搜尋時，卻會搜尋到其他城市的商品，因此我們的 PO 提出城市內搜尋的功能，透過團隊內部的討論，我們最後使用開關的方式讓使用者可以切換搜尋範圍。
